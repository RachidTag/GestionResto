#Présentation du projet
Notre idée principale est de développer une application utile aux restaurateurs et à leurs employés.

Notre objectif est de créer un outil proposant le plan interactif d’un restaurant (pouvant posséder plusieurs salles). Sur celui-ci on pourra observer l’agencement des tables dans la salle.

1) Le point de vue « chef de salle »

Via des icônes sur les tables, nous pourrons vérifier si celles-ci sont libres, réservées ou occupées.

En sélectionnant une table nous pourrons voir différentes informations sur celle-ci. Par exemple à quelle heure les clients sont arrivés (ou doivent arriver pour une réservation), si leur serveur s’est occupé d’eux. Nous aurons une barre de progression pour voir où ils en sont dans leur repas, et quand celle-ci sera arrivée au bout nous aurons un message pour dire qu’on peut emmener l’addition.
Sur la page principale le chef de salle pourra voir, pour chaque secteur, comment est chargé le serveur (nombre de clients total à gérer, à prendre en charge…) afin de connaître l’endroit optimisé où placer les futurs clients.
Enfin, le chef de salle aura un outil automatique pour savoir comment le mieux placer les clients (si la salle est presque pleine on pourrait calculer le temps d’attente). On aurait la possibilité aussi de savoir quelles tables à déplacer pour faire une table adéquat (on veut une table de 10 on a une table de 5 et une autre table de 5 à assembler).

2) Le point de vue « serveur »

Chaque serveur pourra voir les nouvelles tables à prendre en charge, l’ordre d’arrivée, le temps d’attente de chaque client. Une fois que la prise en charge est effectuée il pourra voir si leurs plats sont préparés. A chaque fois qu’il avance dans sa prise en charge il l’indique afin d’avertir le chef de salle de l’avancée de la table. Quand une table attend depuis trop longtemps il recevra des alertes.
Le système ne gèrera pas la prise de commande (quels produits le client choisit) mais la gestion de son secteur pour optimiser son travail. Cependant dans un futur on pourrait ajouter cette fonctionnalité pour avoir un outil optimal pour la restauration.
